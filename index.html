Project

開発プロジェクト

個人開発 本番運用中 卒業研究

患者・医療従事者役 AI シミュレーター

薬学生の実務実習 OSCE 対策向け、AI ロールプレイ練習ツール

概要

薬学部 5 年生の OSCE・実務実習に向けて開発。Google Gemini 2.5 Flash が患者・医師・看護師を演じ、 実際のコミュニケーション練習ができる。会話終了後に AI がチェックリストに基づいた評価フィードバックを返す。 Streamlit Community Cloud で本番運用しており、複数ユーザーに利用されている。

主な機能

🎭 AI による患者・医師・看護師ロールプレイ（10 種類以上のシナリオ）
📊 チェックリスト + SOAP 薬歴の AI 自動評価（JSON パース）
🎓 OSCE 対策 / 実務実習前練習 / 初期研修 の 3 モード
🔐 Supabase Auth による認証・自動ログアウト（30 分）
🔊 gTTS による AI 音声読み上げ
👩‍🏫 教員向けスタッフダッシュボード（全学生評価の一覧閲覧）

技術スタック

Google Gemini 2.5 Flash
Streamlit 1.41
Supabase (Auth + PostgreSQL)
Python 3.x
gTTS
Streamlit Community Cloud

アーキテクチャ

ユーザー
↓
Streamlit UI (app.py)
├── 認証 (auth.py / Supabase Auth)
├── シナリオ選択 (sidebar.py / prompts*.py)
├── チャット (ui_chat.py / llm.py)
│ └── Gemini 2.5 Flash API
├── 音声 (audio.py / gTTS)
└── 評価 (evaluation.py)
└── Supabase DB (evaluations テーブル)

工夫した点

① プロンプトで「聞かれたことだけ答える」制約を設け、実臨床に近い会話を再現
② 評価を JSON 構造化出力させ、チェックリスト・アドバイス・SOAP をパース・表示
③ last_active_at を Supabase に記録し、サーバー再起動後もタイムアウト判定が動作
④ {{TODAY+3Y}} などの日付テンプレートを起動時に実日付へ動的置換
